01 — Visual Theme & Atmosphere

Darkness as
engineered medium.

Linear's website is a masterclass in dark-mode-first product design — a near-black canvas where content emerges like starlight. Information density is managed through gradations of white opacity rather than color variation. This is not a dark theme applied to a light design; it is darkness as the native medium.

Dark canvas

Marketing black (#08090a) as the deepest surface, with an imperceptible blue-cool undertone. Panel dark (#0f1011) one step up. Near-white text at #f7f8f8 — not pure, preventing eye strain.

Whisper borders

Semi-transparent white at rgba(255,255,255,0.05) to rgba(255,255,255,0.08) — wireframes drawn in moonlight. Structure without visual noise.

Inset + drop shadows

Dark surfaces get a subtle inset 0 1px 0 rgba(255,255,255,0.03) top highlight paired with dark drop shadow below. Depth from luminance, not just layering.

Variable Inter at weight 510

Inter Variable with cv01 + ss03. Signature weight 510 sits between regular and medium — subtle emphasis that doesn't shout. Display sizes use aggressive negative tracking (−1.584px at 72px).

04 — Typography Hierarchy

Fifteen roles,
one variable family.

Inter Variable from weight 300 to 590. The signature weight is 510 — between regular and medium, creating subtle emphasis that doesn't shout. Display sizes (72/64/48px) use aggressive negative letter-spacing (−1.584px at 72px) for an engineered feel.

Font note

Linear uses Inter Variable with OpenType features cv01 (single-storey a) and ss03 (geometric letterform variants) enabled globally. Variable font loading (wght@100..900) gives access to weight 510 — otherwise only discrete weights (400/500) are available. Berkeley Mono is Linear's private monospace (paid/SIL license); this board falls back to JetBrains Mono. Anywhere the spec says Berkeley Mono, use font-family: 'JetBrains Mono', ui-monospace, SF Mono, Menlo.

05 — Typography Principles

Compression scales
with size.

The bigger the type, the tighter it gets. Four rules govern the entire system.

Compression at scale

72px → −1.584px · 64px → −1.408px · 48px → −1.056px · 32px → −0.704px · 24px → −0.528px · 16px → normal. Headlines feel engineered — body text breathes.

Weight 510 signature

Inter Variable's 510 sits between 400 (regular) and 500 (medium). Used for most UI text, display headings, labels, and micro-text. Creates consistent emphasis without the weight jumps of discrete-weight systems.

Tight display, loose body

Line height contracts as size grows: 1.55 at body → 1.17–1.20 at heading → 1.00 at display. Denser headlines create authority; breathing body copy preserves readability.

Label / Micro positive tracking

At 13px Label and 11px Micro, letter-spacing opens up to +0.02em and +0.04em respectively. Small text gets more room to breathe; large text gets compressed.

18 — Iteration Guide

Eight rules you
can't violate.

Print this list and tape it to your monitor. If you break any of these, it stops being Linear.

Dark is the canvas. #08090a for marketing, #0f1011 for panels. Never white, never light-mode by default.
Inter Variable with "cv01","ss03" globally. Weight 510 for most UI text, not 400 or 500.
Letter-spacing scales with font size: −1.584px at 72px · −1.056px at 48px · −0.528px at 24px · normal at 16px · positive at 13px Label (+0.02em) and 11px Micro (+0.04em).
Borders are semi-transparent white: rgba(255,255,255,0.05) default, 0.08 for stronger separation. Never solid black or solid white borders.
Elevation uses inset top highlight + dark drop shadow. Inset creates the edge, drop creates the depth. Never plain drop shadow alone — it disappears on dark.
No warm/cool section alternation. The page is one unbroken dark canvas. Hierarchy comes from card surface (level-3 #191a1b vs canvas #08090a), not bg swaps.
Button radius is 6px, panel radius is 8px, modal radius is 12px. Never larger than 12px — Linear is sharp.
Brand indigo (#5e6ad2) is the only chromatic color in core UI. Status green/emerald appears only on progress indicators. Use accents sparingly.

Rules of the name

Semantic over visual. Use --color-text-primary, not --color-black-95. Visual names break when the brand shifts; semantic names survive.
State is a suffix, not a prefix. --color-accent-hover (good). --hover-color-accent (bad).
No component names in base tokens. --button-bg is a component token. It should reference a base: --button-bg: var(--color-accent). Two-layer system — never hardcode.
Scale uses numbers, role uses words. Spacing/radius follow --space-8. Semantic roles follow --color-text-secondary. Don't mix.

Why four surfaces per state on dark?

On dark canvas, a single hex can't cover all four roles — the same green that works for an icon turns muddy as a text color on a green-tinted surface. Linear uses semi-transparent tints (rgba(…,0.14) bg / 0.42 border) plus a lightened text variant to guarantee readability without losing the semantic signal.

21 — Motion Tokens

Three durations.
Three curves.
That's it.

Linear avoids excessive motion — UI feedback is fast and subtle. Lock three durations (120/200/400ms) and three easing curves upfront. Override only with a clear reason.

Duration scale

120ms · fast

Micro-interactions

Button hover, icon state, input focus ring. Should feel immediate — no lag, no animation.

200ms · base

Standard transitions

Card hover lift, modal open, tab switch, dropdown reveal. The default for almost everything.

400ms · slow

Content reveal

Page transitions, large panel slides, hero illustration entrances. Anything asking for attention.

Easing curves

cubic-bezier(0.2, 0, 0, 1)

Ease-out

Default for UI. Fast start, soft stop. Feels responsive.

cubic-bezier(0.4, 0, 0.2, 1)

Ease-in-out

Soft both ends. Modal opens, page transitions.

cubic-bezier(0.34, 1.56, 0.64, 1)

Ease-spring

Slight overshoot. Playful micro-delights only. Rare.

Respect reduced-motion

Wrap non-essential motion in @media (prefers-reduced-motion: reduce). Duration → 0ms, transforms → none. Essential feedback (focus rings, keyboard press) stays.

06 · Do & Don't
✓ Do
  • One primary button per view
  • Action-oriented label ("Send package", not "OK")
  • Show loading state while async work runs
  • Use destructive variant for delete / discard
✗ Don't
  • Don't stack two primary buttons side by side
  • Don't use ghost button for the main action on a page
  • Don't disable without an obvious reason ("Why is this gray?")
